Adapter seam — rozetka adapteri

Agentlar tashqi tizimga to'g'ridan-to'g'ri emas, adapter orqali ulanadi. Bugun fake/local (offline), ertaga real MCP — bitta almashtirish, logika tegmaydi.

FAKE

In-memory

Issue/MR/CI hammasi .json faylda. Test + demo. Hozir ishlaydi.

LOCAL

Real, credsiz

Haqiqiy git + haqiqiy node --test sandbox'da. Kod chindan quriladi.

REAL

Live MCP

Plane / GitLab / Docmost / Mailcow. Creds kelganda HERMES_MODE=real.

Non-tech tushuntirish: telefon zaryadkasi bir xil — faqat rozetka adapteri o'zgaradi (UK/EU/US). Bizning "telefon" = pipeline; "adapter" = qaysi tizimga ulanish. Telefon o'zgarmaydi.

Pros va Cons

✅ Afzalliklar

  • Tezlik — 80% ish avtomatik, odam faqat qarorda
  • Self-hosted — SaaS to'lov yo'q, ma'lumot o'zingizda
  • Adapter seam — fake→real bitta almashtirish
  • TDD majburiy — har kod testdan o'tadi
  • Xavfsiz — merge/prod doim inson nazoratida
  • Kuzatiluvchi — log + monitor + eskalatsiya, sokin fail yo'q
  • Offline isbotlangan — creds'siz to'liq ishlaydi

⚠️ Kamchiliklar / cheklovlar

  • Token narxi — parallel agentlar qimmat
  • Self-host yuki — server, backup, uptime sizda
  • Real MCP hali ulanmagan — creds kutadi
  • Sessiya ochiq (Path A) — 24/7 emas, hali
  • AI sifati — murakkab ishda inson review shart
  • Yangi tizim — jamoa o'rganishi kerak
